# 钉钉发送 Word 文件成功方法

**创建时间**: 2026-03-10 13:12  
**场景**: 需要在钉钉群聊中发送 Word/PDF/PPT 文件附件

---

## ✅ 成功的方法

### 方法 1: 使用 OpenClaw message 工具（推荐）⭐⭐⭐⭐⭐

**适用场景**: 在当前钉钉群聊中发送文件

**代码示例**:
```python
# 在 Python 脚本中调用 message 工具
from openclaw import message

message(
    action="send",
    channel="dingtalk",
    filePath="/path/to/your/file.docx",
    message="📄 Word 文档已发送\n\n**文件**: 文件名.docx\n**大小**: XX KB",
    target="群聊 conversationId"
)
```

**优点**:
- ✅ 简单直接
- ✅ 支持文件附件
- ✅ 直接发送到群聊
- ✅ OpenClaw 自动处理

**缺点**:
- ❌ 需要 OpenClaw 环境

---

### 方法 2: 邮件附件（备用）⭐⭐⭐⭐

**适用场景**: 文件较大或需要正式发送

**代码示例**:
```python
import smtplib
from email.mime.multipart import MIMEMultipart
from email.mime.base import MIMEBase
from email import encoders

# 配置
SMTP_SERVER = "smtp.qq.com"
SMTP_PORT = 465
SENDER_EMAIL = "your_email@qq.com"
SENDER_PASSWORD = "your_auth_code"
RECEIVER_EMAIL = "receiver@qq.com"

# 创建邮件
msg = MIMEMultipart()
msg['From'] = SENDER_EMAIL
msg['To'] = RECEIVER_EMAIL
msg['Subject'] = "📄 文件发送"

# 添加附件
with open("/path/to/file.docx", 'rb') as f:
    attachment = MIMEBase('application', 'vnd.openxmlformats-officedocument.wordprocessingml.document')
    attachment.set_payload(f.read())
    encoders.encode_base64(attachment)
    attachment.add_header('Content-Disposition', 'attachment', filename="file.docx")
    msg.attach(attachment)

# 发送邮件
server = smtplib.SMTP_SSL(SMTP_SERVER, SMTP_PORT)
server.login(SENDER_EMAIL, SENDER_PASSWORD)
server.send_message(msg)
server.quit()
```

**优点**:
- ✅ 稳定可靠
- ✅ 支持大文件
- ✅ 正式正式

**缺点**:
- ❌ 需要切换到邮箱查看

## ❌ 失败的方法

### 1. 钉钉机器人 webhook 直接发送文件
- ❌ webhook 只支持文本/Markdown/图片
- ❌ 不支持文件附件
- ❌ body 大小限制 20KB

### 2. 钉钉工作通知 API 发送文件
- ⚠️ API 返回成功但实际收不到
- ⚠️ 需要正确的用户 ID
- ⚠️ 发送到工作通知，不是群聊

### 3. 群会话 API
- ❌ v1.0/robot/groupMessages/send 返回 404
- ❌ 参数不合法或会话 ID 无效

## 📋 最佳实践

### 场景 1: 在当前群聊发送文件
**推荐**: OpenClaw message 工具
```python
message(
    action="send",
    channel="dingtalk",
    filePath="/path/to/file.docx",
    message="文件说明",
    target="群聊 conversationId"
)
```

### 场景 2: 发送给个人
**推荐**: 邮件附件 或 工作通知 API
- 邮件：稳定可靠
- 工作通知：需要正确的用户 ID

### 场景 3: 发送大文件（>50MB）
**推荐**: 邮件附件 或 云存储链接
- 邮件：QQ 邮箱支持超大附件
- 云存储：阿里云 OSS、腾讯云 COS

## 🔑 关键配置

### 钉钉群 conversationId
- 格式：`cidXXXXXXXXXXXXXXXXX==`
- 获取方式：从消息元数据中获取
- 示例：`cidMDsc67KFuz8hnEEhjwwbKQ==`

### 钉钉 webhook
- 格式：`https://oapi.dingtalk.com/robot/send?access_token=XXX`
- 用途：发送文本/Markdown 消息
- 限制：不能发送文件附件

### 钉钉工作通知 API
- Token: 通过 app_key + app_secret 获取
- API: `topapi/message/corpconversation/asyncsend_v2`
- 用途：发送工作通知（个人）
