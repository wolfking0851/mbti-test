## 🎯 核心架构

**问题**: Gateway 配对问题导致无法使用 sessions_send 跨对话框调度

**解决方案**: Python 脚本 + 通知文件 + 七娃 Gateway Heartbeat

```
【任务执行层】
Python 脚本（每 5 分钟）
├── 检查 /workspace/team-tasks/
├── 发现任务 → 调用阿里云执行
├── 执行完成 → 写入通知文件
└── 资源：50MB 内存，¥0.5/月

【AI 回复层】
七娃 AI（Gateway Heartbeat，每 30 分钟）
├── 检查 /workspace/team-tasks/notifications/
├── 发现未读通知 → 直接对话框回复老板
└── 资源：0（Gateway 内置）
```

- `/workspace/scripts/heartbeat-checker.py` - Python 检查脚本（已添加通知功能）

## 💡 重要学习

### 本地模型不需要！❌

**最初误区**: 以为需要本地模型 (Ollama + Qwen2.5-0.5B) 做 Heartbeat 检查

**真相**: Heartbeat 检查只是字符串匹配，不需要 AI 能力

**资源对比**:
- Python 脚本：50MB 内存，¥0.5/月
- 本地模型：550MB 内存，¥1.5/月

**决策**: 本地模型已安装但不用（留着未来可能有用）

### 七娃配置要点

**不要说技术细节**（Python、systemd、Cron）
**只说必要信息**:
- 检查哪里（通知目录）
- 检查什么（未读通知）
- 怎么回复（直接对话框回复）
