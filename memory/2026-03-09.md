### 核心技术发现

**1. 视频生成必须用 multipart/form-data** ⭐⭐⭐
```bash
# ❌ 错误（JSON 格式）
curl -d '{"files":["image.png"]}' ...

# ✅ 正确（multipart/form-data）
curl -F "model=seedance-2.0" \
     -F "prompt=@1 提示词" \
     -F "files=@图片.png;type=image/png" \
     -H "Authorization: Bearer SessionID"
```

**2. @引用系统**
- `@1` `@2` `@3` 在 prompt 中引用上传的图片
- 支持单图（首帧生视频）
- 支持多图（全能参考，最多 3 张）
- 支持首尾帧（2 张图，开始 + 结束）

**3. 视频生成超时问题（-2007）**
- **原因**：curl 等待时间不够（Seedance 2.0 需要 5-10 分钟）
- **现象**：curl 返回超时错误，但 API 后台继续轮询
- **解决**：后台提交（nohup），网页端监控进度

**4. 高频率轮询导致限流** ⭐⭐⭐
- **问题**：API 每秒轮询 1 次，8 个镜头 = 8 次/秒
- **后果**：即梦服务器限流，视频生成变慢甚至卡住
- **解决**：降低轮询频率，或去网页端查看进度

## 💡 新经验教训

### 即梦 AI 视频生成

1. **视频生成必须用 multipart/form-data**
   - JSON 格式会报错"需要至少一个文件"
   - 使用 `-F` 参数而不是 `-d`
   - `files` 参数名必须正确

2. **后台提交视频任务**
   - 使用 `nohup curl ... &`
   - 避免 curl 超时（Seedance 2.0 需要 5-10 分钟）
   - 去网页端监控进度

3. **避免高频率轮询**
   - API 默认 1 秒 1 次轮询会导致限流
   - 8 个镜头同时轮询 = 约 57,600 次/2 小时
   - 解决：网页端查看进度，不依赖 API 轮询

4. **图片下载要及时**
   - 即梦图片 URL 有时效性
   - 生成后立即下载保存

5. **提示词要详细**
   - 使用 8 维度公式
   - 包含约束条件（避免变形）
   - 明确镜头运动（推近/拉远/横摇）

### 钉钉文档发送

1. **使用 message 工具最简单**
   - 不需要复杂配置
   - 支持所有常见文档格式
   - 自动处理文件上传

2. **需要群 ID**
   - 从钉钉群设置获取
   - 格式：`cidPd/xxxxx==`
   - 在 TOOLS.md 中记录

3. **附带说明文字**
   - 说明文档内容
   - 列出关键点
   - 便于接收者快速理解
