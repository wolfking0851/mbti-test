## 🔐 Gateway 操作安全规则（2026-03-09 新增）

### 绝对规则

**任何 Gateway 相关操作前必须**：

1. ✅ **先汇报** - 清楚说明要做什么、为什么、有什么风险
2. ✅ **等确认** - 得到老板明确同意（"同意"、"开始"、"可以"）后再执行
3. ✅ **分步汇报** - 每完成一步都汇报进度
4. ✅ **保持透明** - 随时让老板知道当前状态

### 需要征得同意的操作

| 操作 | 风险等级 | 必须同意 |
|------|---------|---------|
| **停止 Gateway** | 🟡 中（会掉线） | ✅ 必须 |
| **启动 Gateway** | 🟢 低 | ✅ 必须 |
| **重启 Gateway** | 🟡 中（会掉线） | ✅ 必须 |
| **升级 OpenClaw** | 🔴 高（可能失败） | ✅ 必须 |
| **修改配置** | 🟡 中（可能影响功能） | ✅ 必须 |
| **安装插件** | 🟢 低 | ✅ 必须 |

### 违规后果

- ❌ 未经同意操作 Gateway → 严重违规
- ❌ 操作后不汇报进度 → 严重违规
- ❌ 重复回复相同通知 → 需要优化

### 通知去重规则

**避免重复回复相同的 Gateway 通知**：

- ✅ 相同类型的通知，5 分钟内只汇报一次
- ✅ 如果连续收到相同通知，合并成一次汇报
- ✅ 使用"继续汇报"而不是重复完整内容

### 📦 备份保留规则（2026-03-09 新增）

**每日备份文件保留策略**：

- ✅ **只保留最新的 2 个备份**
- ✅ 自动删除超过 7 天的旧备份

- 或在 Cron 中设置定期清理任务
