## 📁 文件组织

### 命名约定
- 日期格式：`YYYY-MM-DD`
- 身份编号：`SF-0001`（4 位数字）
- 任务编号：`task-0001`（4 位数字）

### 存储位置
```
/workspace/
├── life/              # 3 层记忆系统
├── memory/            # 每日记忆
├── tasks/             # 任务记录
├── identity/          # 身份档案
├── skills/            # 技能
└── reports/           # 报告
```

## 🔐 安全规则

### Gateway 操作
- ⚠️ **必须征得同意**
- ✅ 先汇报 → 等确认 → 再执行
- ✅ 分步汇报进度

### 备份策略
- 保留最新 2 个备份
- 自动删除超过 7 天的旧备份
- 保留至少 1 个完整备份

---

## 📊 汇报机制

### 每日复盘
- **时间**: 04:00（自动）
- **内容**: 工作总结、学习进化、问题反思、明日计划
- **格式**: Markdown 报告

### 项目汇报
- **触发**: 阶段完成或里程碑
- **内容**: 进度、问题、下一步
- **渠道**: 钉钉 + 邮件

## 📝 血泪教训（已编码为习惯）

1. **身份命名** - 不要有空格（身份 -0001 ✓）
2. **脚本依赖** - 必须包含智能检查
3. **任务停滞** - 30 分钟主动检查
4. **进度汇报** - 核实最新 state.json
5. **Gateway 操作** - 必须征得同意
6. **通知去重** - 5 分钟内只汇报一次
7. **备份保留** - 只保留最新 2 个
